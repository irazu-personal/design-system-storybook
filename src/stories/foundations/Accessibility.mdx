# Accessibility

Spark components follow WCAG-oriented patterns: visible focus, sufficient contrast on primary actions, and disabled states that do not rely on color alone.

## Focus ring

Primary action elements use a token-backed focus ring (`semantic/action/primary/border/focus`). Tab to the interactive button to see the ring update on focus.

## Contrast

- Primary button text uses `--semantic-button-primary-content-on-surface` on `--semantic-button-primary-bg-default`.
- Secondary and tertiary variants maintain readable contrast on default surfaces.
- Danger variants use dedicated semantic danger tokens.

## Disabled state

Disabled buttons set `pointer-events: none`, use `--semantic-button-disabled-bg` and `--semantic-button-disabled-content-on-action`, and should include `aria-disabled` or the native `disabled` attribute.
